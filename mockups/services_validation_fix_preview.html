Why "services selected but error persists": toggleService() updates _svcSel and re-renders tiles correctly. But checkReq() crashes before it can clear the error.

Law F1 (captured post-commit to Drive):
Form validation must read from the same source the UI writes to. If UI updates a Set/Map, validation reads from that Set/Map. If UI updates DOM, validation reads from that DOM. Never two variables tracking the same value, never a derived display string as a validation signal.

Three bugs today trace to this pattern:
① Driveway material always-required — wrong source in required check
② How-heard false mandatory — useMatch didn't write to nLeadSource DOM; validation read DOM
③ Services/roof — validation read derived fullDescription instead of _svcSel; derivation threw ReferenceError
